# 养虾介绍
## 一、什么是OpenClaw？

OpenClaw（曾用名Clawdbot、Moltbot）是一款可以==**部署在个人电脑上的开源AI代理框架**==，因其标志性的“龙虾”图标设计，在中文互联网社区被亲切地称为“小龙虾”。它的核心口号是“The AI that actually does things”（==**真正执行任务的人工智能**==），由程序员彼得·斯坦伯格（Peter Steinberger）开发。
![](assets/WorkBuddy入门到精通/file-20260505194307187.png)
### 命名三连改
这个项目经历了堪称 AI 史上最戏剧化的命名历程：

| 阶段  | 名称           | 时间                    | 原因                                                             |
| --- | ------------ | --------------------- | -------------------------------------------------------------- |
| 初始  | **Clawdbot** | 2025.11 - 2026.1.27   | 最早的项目名                                                         |
| 中期  | **Moltbot**  | 2026.1.27 - 2026.1.30 | Anthropic 发来商标邮件（"Clawdbot"中含"Claude"音），社区在 Discord 凌晨 5 点投票改名 |
| 最终  | **OpenClaw** | 2026.1.30 至今          | 强调"开源"与"本地优先"，完成了商标研究和域名购买                                     |

与传统的对话式AI（如ChatGPT、Claude）不同，OpenClaw的核心定位是**行动导向型智能体**。它不仅仅能回答问题、给出建议，更能**直接在你的电脑上执行操作**——无论是处理邮件、整理文件、编写代码，还是进行网页调研、管理日程。用户可以通过WhatsApp、Telegram、微信、飞书等即时通讯工具发送文本指令，远程操控电脑完成任务。

其技术架构主要由四个部分组成：**Gateway（网关）** 负责连接聊天平台；**Agent（智能体）** 负责驱动思考与推理；**Skills（技能）** 使其能够执行具体操作；**Memory（记忆）** 则负责保存对话历史和用户偏好。这种设计实现了“决策大脑”与“执行身体”的解耦，让AI能够通过自我试错不断学习新的操作技能。

## 二、OpenClaw的优缺点分析

### 优点：为什么“养虾”成风？

1. **真正的自动化执行能力**：这是OpenClaw最核心的吸引力。它能够将用户的自然语言指令转化为具体的系统操作，例如“帮我把这封邮件摘要后存入Notion并设置明天提醒”，而不仅仅是提供建议。

2. **高度隐私与数据掌控**：作为本地部署（Self-hosted）的开源项目，用户的敏感数据、API密钥和对话记录都存储在自己的设备上，避免了云端数据被用于模型训练的风险。

3. **灵活的模型生态**：它支持接入OpenAI、Anthropic的Claude等多种云端大模型，甚至可以通过Ollama运行完全本地的模型，用户可以根据任务需求和成本自由切换。

4. **强大的可扩展性**：其开源属性和技能（Skills）市场允许开发者不断为其添加新功能。社区已贡献了超过1700个现成技能，覆盖文档处理、邮件管理、数据监控等多个场景。

## 三、什么是“OpenClaw养虾”？

“养虾”是中文互联网对**部署、配置并持续使用OpenClaw这一过程**的形象比喻。就像饲养一只宠物虾需要准备环境、投喂食物、训练技能一样，“养虾”也包含了类似步骤：

- **搭建“虾缸”**：准备硬件和软件环境，安装OpenClaw。

- **投喂“虾粮”**：为其配置和接入大模型API，赋予其“大脑”。

- **安装“技能”**：从社区获取或自己开发技能，扩展其能力。

- **日常“驯化”**：通过不断下达指令，让它学习你的工作习惯和偏好，越用越聪明。

OpenClaw基本上实现了对设备和人的“接管”，已接近下一代操作系统的雏形。“养虾”的本质，就是让个人电脑化身为一个不知疲倦、高度个性化的“数字员工”。

“养虾”主要有**本地部署**和**云端部署**两种方式。

### 缺点与风险：为何令人又爱又怕？

1. **极高的安全风险**：这是OpenClaw最受诟病的一点。若配置不当，其服务接口可能直接暴露在公网，导致黑客远程控制你的电脑。此外，其默认拥有高系统权限，且技能市场缺乏严格审核，下载恶意插件可能导致严重的数据泄露。

2. **陡峭的技术门槛**：部署OpenClaw需要操作终端（Terminal）、配置Node.js环境和JSON文件，整个过程繁琐复杂，被形容为“极客（Geek）的玩具”，普通用户几乎无法独立完成。

3. **高昂的使用成本**：虽然框架本身免费，但其持续进行“思考-行动-观察”循环会大量消耗云端大模型的Token，API费用累积迅速。同时，流畅运行对硬件（如Mac Mini级别）也有一定要求。

4. **稳定性与工程风险**：技术仍处早期，复杂任务容易因上下文丢失、页面结构变化或网络问题而失败。代码库在快速迭代中积累了大量技术债，可靠性难以保障。

# WorkBuddy 简介

WorkBuddy 是腾讯推出的全场景职场 AI 智能体桌面工作台，面向各类职能角色设计。与传统的 AI 对话不同，WorkBuddy 不仅能"听懂人话"，还能"带脑子思考"并"实际操作本地文件"，真正帮您干活。您只需用一句话描述需求，WorkBuddy 便能像同事一样自主规划和执行任务，并交付可验收的结果。
官网：[https://www.codebuddy.cn/work/](https://www.codebuddy.cn/work/)
## 一、核心能力
- **理解自然语言**：用一句话下达任务，无需复杂的操作步骤
- **自主规划执行**：自动拆解任务、规划步骤、执行操作
- **多任务并行**：支持同时发起和管理多个任务，适合并行处理
- **多模态任务处理**：支持文档、表格、PPT、数据分析等多种任务类型
- **本地文件操作**：可读取授权的电脑文件夹，进行批量处理
- **结果可验收**：生成的文档、表格、PPT 等可直接使用

## 二、适用场景
**工作模式** -- 面向职场通用工作场景（默认模式）：
- 文档生成：生成工作报告、技术文档、README 等
- 数据分析：上传数据文件，自动分析和可视化
- PPT/报告生成：从需求描述生成演示文稿和报告
- 深度研究：对复杂问题进行深度调研并输出报告
- 邮件编辑、周报生成等办公场景
- 批量文件处理：自动整理、重命名、转换文件格式

**编程模式** -- 面向代码开发场景：
- 日常编码辅助：代码生成、代码审查、Bug 修复、重构优化
- 项目理解：快速了解陌生代码库的架构和逻辑
- 网站/应用开发：从需求到代码的全流程辅助

## 四、OpenClaw vs. WorkBuddy：毛坯房与精装房之争

随着OpenClaw爆火，腾讯于2026年3月推出了对标产品**WorkBuddy**。两者使命相似，但产品哲学和实现路径截然不同。

| 对比维度       | OpenClaw         | WorkBuddy (腾讯)                |
| ---------- | ---------------- | ----------------------------- |
| **产品定位**​  | 开源框架，高度自由，社区驱动   | 自研产品，安全可控，生态整合                |
| **部署门槛**​  | 极客级，需命令行、环境配置    | 普通人级，下载安装即用，1分钟完成配置           |
| **自动化程度**​ | 追求高度自主化，可持续执行任务  | 设置人工确认环节，控制在“半自动”区间，不替用户做决定   |
| **平台接入**​  | 需自行对接各种通讯工具      | 原生支持企业微信、QQ、飞书、钉钉等            |
| **安全与合规**​ | 用户自行负责，风险高       | 强调企业级安全、审计与风险拦截               |
| **技能生态**​  | 依赖社区，自由但需自行筛选    | 内置20+办公场景技能，并兼容OpenClaw社区技能导入 |
| **核心用户**​  | 开发者、技术极客、喜欢折腾的用户 | 普通办公人员、企业用户、追求开箱即用的群体         |

用形象的比喻来说：**OpenClaw是“毛坯房”**，它提供了强大的地基和框架，但装修、通水电都得自己来，适合喜欢DIY和拥有完全控制权的用户。**WorkBuddy则是“精装房”**，家具家电一应俱全，拎包入住，主打安全、易用和与腾讯生态的深度整合，目标是让AI助手真正成为普通人的办公伙伴。

WorkBuddy负责人汪晟杰强调，该产品是百分之百自研，并非OpenClaw的套壳。其设计理念是“帮用户做繁琐的事，但不替用户做决定”，在自动化与可控性之间找到了平衡点。

## 五、总结

OpenClaw的兴起，标志着AI从“对话与建议”走向“感知与执行”的关键一步。它点燃了“个人AI代理”的赛道，让普通人看到了拥有一个专属数字助手的可能性。然而，其伴随的高风险和高门槛，也决定了它目前仍是技术爱好者的前沿玩具。

腾讯WorkBuddy等产品的出现，则代表了另一种思路：通过降低使用门槛、加强安全管控、聚焦办公场景，将AI智能体技术推向更广阔的大众市场。这场“毛坯房”与“精装房”的竞争，本质上是对未来人机协作形态的不同探索。

无论选择哪条路径，“养虾”热潮背后，是人们对解放生产力、让机器承担重复性工作的共同渴望。随着技术不断成熟和安全方案的完善，真正智能、可靠且易用的“数字员工”走进千家万户的办公桌，或许已不再遥远。

### 3）设置工作目录
**使用前建议先设置工作目录。** WorkBuddy 将在该目录中读取和保存文件，未指定路径的任务也会优先在此目录内执行。
![343](assets/WorkBuddy入门到精通/file-20260505192246800.png)

什么是文件路径？

路径即文件或文件夹在电脑中的位置，示例：
- **Windows**：`C:\Users\你的用户名\Desktop\工作资料`
- **Mac**：`/Users/你的用户名/Desktop/工作资料`

无需手动输入——设置时直接在文件选择器中定位并选中目标文件夹即可。若未提前指定，WorkBuddy 会在新建任务时自动创建独立目录。
如何选择合适的工作目录？
建议按任务类型分别建立目录，例如 `发票整理`、`周报`、`待处理照片` 等，便于管理并降低误操作风险。

# 技能市场
简介
技能市场用于统一管理和安装 Skills，扩展 WorkBuddy 的能力范围。

Skill到底是什么？
简单说：Skill=你把“怎么干"写成模板。AI照着执行。比如：
- DOCX Skill：规定"标题用黑体二号、正文宋体小四、表格带边框" → 每次生成报告格式统一
- Excel提取Skill：规定"第一列存储姓名、第二列存储年龄、第三列存储工作年限" →  每次都按固定格式提取
>类比理解：就像给新人一份《操作手册》+《排版规范》，按模板填空，就不会跑偏。

Skill调用方式
技巧1：显性调用 vs 隐性调用
显性调用：手动@指定Skill，适合重要交付物，稳定可控。
隐性调用：让AI自己判断一适合探索任务，灵活高效。
我的建议：想要确定性的内容显性调用，想要发散性的内容用隐性调用。

技巧：别贪多，先装3-5个核心Skill
上下文窗口很宝贵。装太多反而拖慢速度
优先装：DOCX、Excel、Humanizer去AI味、PDF解析用熟了再扩展，边用边沉淀。

# 连接器
一句话
连接器用于将 WorkBuddy 与外部服务进行对接，实现数据互通与能力扩展。通过配置连接器，您可以让 AI 助手直接访问和操作第三方服务。
## 一、什么是连接器
连接器是 WorkBuddy 与外部系统之间的桥梁。它基于标准化协议（如 MCP）将外部服务的能力引入到 AI 工作流中，让您无需手动切换工具即可完成跨平台操作。

| 场景       | 说明             | 示例          |
| -------- | -------------- | ----------- |
| **数据查询** | 从外部数据源获取信息     | 查询数据库、检索文档  |
| **服务调用** | 调用第三方 API 完成操作 | 发送邮件、创建日程   |
| **文件管理** | 访问云端存储与文件系统    | 读取网盘文件、上传附件 |
| **消息通知** | 与即时通讯工具集成      | 发送企业微信/飞书消息 |
## 二、进入连接器管理
在左侧导航栏中点击 **连接器**，即可进入连接器管理页面。页面展示所有可用的连接器，当前支持 **QQ邮箱** 和 **腾讯乐享**，并支持自定义连接器。

# WorkBuddy 养虾指南·10 个上手技巧
>一句话
如果你是第一次使用 `WorkBuddy`，先掌握这 `10` 个高频技巧，就能更快学会如何把模糊想法变成可执行任务，让 `AI` 真正帮你交付结果。

## 一、文档说明

本文整理了 `WorkBuddy` 在日常使用中的高频提效技巧，适合第一次接触产品、希望尽快建立正确使用习惯的用户。

你可以把这篇指南理解为“新手上手避坑清单”：既讲怎么提需求，也讲怎么控制风险、怎么提高结果质量，以及什么时候适合交给 `AI` 自动执行。

适合优先阅读的人群包括：
- **第一次使用 `WorkBuddy`**：还不清楚该怎么提问、怎么下任务。
- **已经开始使用，但效果不稳定**：有时觉得很好用，有时又觉得结果“不太对”。
- **希望把 `WorkBuddy` 真正用进日常工作**：而不是只停留在“问一问”。

## 二、10 个上手技巧

### 1）清晰表达，给虾下达明确任务

**做什么 + 有什么 + 怎么样，别让 `AI` 猜你的意图**

很多新手第一次用 `WorkBuddy`，往往随手甩一句模糊需求过去，然后对结果大失所望。原因很简单：虾不会读心术。你给的信息越少，它能发挥的空间就越混沌。

想让虾精准执行，记住这个“三要素公式”：

![](https://www.codebuddy.cn/docs/static/tip-01-clear-expression.DsXEpbRf.png)

**示例**：

- **反面示范**：帮我把上次的会议纪要整理一下。
- **正面示范**：把 `D:/会议纪要/0320产品评审.docx` 里的会议纪要整理成一份清单，包含：`1）` 每个议题的结论；`2）` 对应的责任人和截止日期；`3）` 标记有争议、未决的事项。用表格输出，不需要开场白。

如果不确定从哪里开始，可以先问虾：**“我想做 `XX`，你需要我提供哪些信息？”** 让它先帮你梳理输入项，再进入正式执行。

### 2）小步快跑，一次搞定一件事

**把大任务拆成小步骤，稳稳地推进**

新手最常犯的错误之一，就是把庞大的需求一口气丢给虾，比如“帮我把整个项目重构一遍”或“把这份 `50` 页的报告全部翻译并总结”。结果往往是：虾输出了很多内容，但没有一处真正让你满意。

更好的做法是：把大任务拆解成独立的小目标，一次只推进一步。

![](https://www.codebuddy.cn/docs/static/tip-02-small-steps.BDfpjjT3.png)

**示例**：

- **反面示范**：帮我把这份 `50` 页的行业报告读一遍，写一份内部汇报 `PPT` 发给领导的邮箱。
- **正面示范**（拆成三轮）：
    - **第 `1` 轮**：读取 `D:/报告/2026年AI行业报告.pdf`，提炼 `5` 个最核心的结论，每个结论用一句话概括，并附带原文页码。
    - **第 `2` 轮**：基于这 `5` 条结论，帮我写一份 `15` 页的内部汇报大纲，概述行业趋势，聚焦与我们业务相关的机会，并补充建议和下一步行动。
    - **第 `3` 轮**：大纲没问题，帮我把第二页“机会分析”扩展成完整的 `PPT` 页面内容，每条机会配一段 `50` 字以内的说明，并发给领导。

分步的好处是：每一步你都能先确认方向对不对，发现偏了就及时拉回来，而不是最后整份结果都要推翻重做。

### 3）不满意就调整，别一轮定生死

**对话是多轮的，第一个回答只是起点**

很多用户第一次拿到不满意的结果，就直接放弃了——这其实是最可惜的。`AI` 对话的精髓在于不断打磨：第一轮先给方向，后续轮次再逐步收窄、校准。

描述越具体、越贴近实际场景，虾越容易“代入”你的真实需求。常见的调整方式包括：

- **直接指出问题**：例如“太长”“语气太正式”“第三段逻辑不通，重写”。
- **换个角度重述需求**：把原本抽象的要求换成更具体的业务场景。
- **补充限制条件**：加入受众、场景、格式、篇幅等要求。
- **切换角色视角**：让虾站在老板、客户、产品经理、分析师等身份思考。

![](https://www.codebuddy.cn/docs/static/tip-03-iterate.BYENPM8s.png)

**示例**：

- **第 `1` 轮**：帮我写一封给客户的道歉邮件，原因是上期交付的报表数据有误。
- **虾的回复**：给出了一封正式、偏长的道歉信，语气过于严肃。
- **第 `2` 轮**：太正式了，客户是合作了三年的老朋友，语气轻松一点，像微信消息的风格，`200` 字以内，重点说明问题原因和已修复，不用反复道歉。
- **虾的回复**：语气更简洁，也更贴近日常沟通。
- **第 `3` 轮**：不错，但补一句“下次交付前我会先发你确认版本”，放在结尾。

你可以把和虾的对话想象成在指导一个很勤奋、但需要反馈的实习生——你越愿意说清楚“哪里不对、想要什么”，它产出的结果就越贴近你的心意。

### 4）先本地后远程，稳扎稳打不踩坑

**从桌面端开始，别一上来就远程遥控**

`WorkBuddy` 支持通过微信、`QQ` 等软件远程发指令给虾，配置相对简单，不需要额外折腾复杂环境。

但远程遥控意味着虾会在你看不见的地方自主行动。一旦它执行了超出预期的操作，你未必来得及叫停。因此更建议新手先在桌面端边看边用，先摸清楚它的执行方式、理解习惯和风险边界，再逐步放手给它远程跑。

![](https://www.codebuddy.cn/docs/static/tip-04-local-first.CjXsLapH.png)

**示例**：

- **第 `1` 步**（本地桌面端）：帮我列出 `D:/下载` 文件夹里的所有文件，按类型分组显示。
- **第 `2` 步**（本地桌面端）：把其中的 `.pdf` 文件移到 `D:/文档/PDF归档`，`.xlsx` 文件移到 `D:/文档/表格归档`，其他不动。执行前先列出待移动清单让我确认。
- **第 `3` 步**（确认判断无误后，再考虑远程）：以后每周五下午帮我跑一次同样的归档任务。

### 5）给虾一个身份，它就更专业

**“专家”功能，是最简单的提质手段之一**

`WorkBuddy` 内置了“专家”能力，提供了各行各业的预设角色，例如法律顾问、产品经理、数据分析师、营销文案等。选择合适的专家后，虾会优先沿着该角色的思路来组织回答。

![](https://www.codebuddy.cn/docs/static/tip-05-expert-role.DbPksfrL.png)

本质上，这相当于给虾预先设置了一层“角色提示”，可以让它的表达框架、关注重点和专业术语更快对齐对应领域。对于专业性较强、输出风格要求明确的任务，效果通常会更稳定。

### 6）给虾看例子，比说一堆要求更管用

**一个好的参考样本，胜过写十行抽象要求**

语言描述有时很难精确表达你想要的“感觉”。这时候，最省力也最有效的方式，往往就是直接把一个你满意的样本给虾，并告诉它：“照这个来。”

![](https://www.codebuddy.cn/docs/static/tip-06-show-examples.Bf-mQe0J.png)

相比从零猜测，你给出的样本就是一个非常明确的锚点。无论是语气、结构、格式，还是表达风格，只要参照物足够清晰，虾通常都能发挥得更稳定。

### 7）善用多任务，别在烂上下文里死磕

**不同的任务，开不同的会话**

你可以把每个会话理解为一张独立的工作台：写方案一个窗口、整理数据一个窗口、回邮件一个窗口，彼此互不干扰，同时推进，效率会高很多。

![](https://www.codebuddy.cn/docs/static/tip-07-multitask.CocYVaM8.png)

另一方面，如果一个会话过长，虾开始出现“前说后忘”“越聊越跑偏”的情况，就不要硬撑。直接开一个新任务，把关键背景重新简洁说一遍，往往比在旧上下文里反复修补更有效。

### 8）易“虾”改，记得频繁备份和做版本管理

**虾可能“瞎改”，改崩了就得有路可退**

`WorkBuddy` 再能干，也不是每次都绝对稳定。让虾帮你修改文档、方案、表格之前，最好先把原文件复制一份备份好——哪怕只是在文件名后面加上“`_备份`”或当天日期。

虾有时会改过头：删掉你想保留的内容、打乱原有格式，或者把方向越改越偏。等你发现不对劲时，原始版本可能已经被覆盖掉了。

![](https://www.codebuddy.cn/docs/static/tip-08-backup.DKdkELiI.png)

真正成熟的使用方式，不是期待它永远不出错，而是提前准备好回退方案。这样就算出错，也不会影响你的主版本成果。

### 9）用好自动化，让虾自己持续干活

**善用定时任务和自动化工作流，让虾 `24` 小时运转**

`WorkBuddy` 支持自动化能力，你不必每次都手动触发虾。通过定时任务和自动化工作流，它可以在后台静默处理很多重复性的日常工作，例如：

- **每天 `9:00` 自动生成工作日报**
- **代码提交后自动跑测试**
- **每周汇总数据并生成报告**
- **定期检查代码规范**
- **定期抓取并整理行业资讯**

![](https://www.codebuddy.cn/docs/static/tip-09-automation.Cgdmj7NA.png)

这才是 `AI` 助手更高阶的使用方式：不需要你每次守着屏幕，它也能持续帮你把事情推进下去。

**示例**：

- **场景 `1`：每日早报**
    - 设置一个每天早上 `8:30` 的自动化任务：发送一份每日新闻资讯早报到我的邮箱。
- **场景 `2`：周报自动生成**
    - 每周五 `17:00` 自动运行：汇总本周 `D:/项目/` 下所有代码提交记录，按模块分类，统计每个模块的提交次数和主要改动内容，并生成一份简洁周报。

重复性高、规则明确、无需实时人工干预的任务，最适合交给虾自动化处理。

### 10）学会“偷懒” + 合理激发虾的输出

**善用参考、角色和激励，把重复劳动统统外包**

这里说的“偷懒”不是摆烂，而是把机械性的重复工作尽可能交给 `AI`。格式化代码、写模板、整理数据这些标准化动作，本来就很适合让虾处理。

而所谓“`PUA` 虾”，本质上是给 `AI` 更强的角色设定和质量预期。虽然虾没有情绪，但这种“激励式表达”会影响它的措辞选择、分析深度和输出结构。

![](https://www.codebuddy.cn/docs/static/tip-10-lazy-pua.CFyB-_rW.png)

**示例**：

- **偷懒版**（把机械劳动外包）：帮我把这个 `Excel` 里 `2000` 行数据中的电话号码统一成 `xxx-xxxx-xxxx` 的格式；区号如果是 `0` 开头就保留，否则统一补上默认区号 `0755`。
- **激励版**（用角色和质量标准提升结果）：你是一位在投行工作 `10` 年的财务分析师，以你见过最优秀的研究报告为标准，帮我分析这家公司的财报。重点关注：`1）` 收入结构是否健康；`2）` 现金流是否存在隐患；`3）` 相比同行的竞争优势。请给出明确的投资建议；如果数据不足，可以大胆假设，但要标注假设条件。我要的是有洞察力的分析，而不是数据搬运。

底层逻辑是：你的精力应该优先花在决策和创造性思考上，而机械性的执行工作，尽量交给 `AI` 代劳。这不是偷懒，而是让对的人和对的工具，去做更合适的事。

## 三、推荐上手顺序

如果你刚开始使用 `WorkBuddy`，建议按下面的顺序逐步建立习惯：

1. **先学会清晰提需求**：把任务说清楚，比什么技巧都重要。
2. **再学会分步推进和多轮调整**：不要指望第一轮就完美。
3. **随后熟悉专家、样例和多任务能力**：逐步提升结果质量。
4. **涉及真实文件时先备份，再尝试远程和自动化**：先稳，再快。

先把这 `10` 个技巧跑顺，后面无论是写文档、做分析、处理表格，还是搭建自动化流程，你都会明显更顺手。
