# 养虾介绍
## 一、什么是OpenClaw？

OpenClaw（曾用名Clawdbot、Moltbot）是一款可以==**部署在个人电脑上的开源AI代理框架**==，因其标志性的“龙虾”图标设计，在中文互联网社区被亲切地称为“小龙虾”。它的核心口号是“The AI that actually does things”，由程序员彼得·斯坦伯格（Peter Steinberger）开发。

与传统的对话式AI（如ChatGPT、Claude）不同，OpenClaw的核心定位是**行动导向型智能体**。它不仅仅能回答问题、给出建议，更能**直接在你的电脑上执行操作**——无论是处理邮件、整理文件、编写代码，还是进行网页调研、管理日程。用户可以通过WhatsApp、Telegram、微信、飞书等即时通讯工具发送文本指令，远程操控电脑完成任务。

其技术架构主要由四个部分组成：**Gateway（网关）**负责连接聊天平台；**Agent（智能体）**负责驱动思考与推理；**Skills（技能）**使其能够执行具体操作；**Memory（记忆）**则负责保存对话历史和用户偏好。这种设计实现了“决策大脑”与“执行身体”的解耦，让AI能够通过自我试错不断学习新的操作技能。

## 二、OpenClaw的优缺点分析

### 优点：为什么“养虾”成风？

1. **真正的自动化执行能力**：这是OpenClaw最核心的吸引力。它能够将用户的自然语言指令转化为具体的系统操作，例如“帮我把这封邮件摘要后存入Notion并设置明天提醒”，而不仅仅是提供建议。
    
2. **高度隐私与数据掌控**：作为本地部署（Self-hosted）的开源项目，用户的敏感数据、API密钥和对话记录都存储在自己的设备上，避免了云端数据被用于模型训练的风险。
    
3. **灵活的模型生态**：它支持接入OpenAI、Anthropic的Claude等多种云端大模型，甚至可以通过Ollama运行完全本地的模型，用户可以根据任务需求和成本自由切换。
    
4. **强大的可扩展性**：其开源属性和技能（Skills）市场允许开发者不断为其添加新功能。社区已贡献了超过1700个现成技能，覆盖文档处理、邮件管理、数据监控等多个场景[web:5]。
    

### 缺点与风险：为何令人又爱又怕？

1. **极高的安全风险**：这是OpenClaw最受诟病的一点。若配置不当，其服务接口可能直接暴露在公网，导致黑客远程控制你的电脑。此外，其默认拥有高系统权限，且技能市场缺乏严格审核，下载恶意插件可能导致严重的数据泄露。
    
2. **陡峭的技术门槛**：部署OpenClaw需要操作终端（Terminal）、配置Node.js环境和JSON文件，整个过程繁琐复杂，被形容为“极客（Geek）的玩具”，普通用户几乎无法独立完成。
    
3. **高昂的使用成本**：虽然框架本身免费，但其持续进行“思考-行动-观察”循环会大量消耗云端大模型的Token，API费用累积迅速。同时，流畅运行对硬件（如Mac Mini级别）也有一定要求。
    
4. **稳定性与工程风险**：技术仍处早期，复杂任务容易因上下文丢失、页面结构变化或网络问题而失败。代码库在快速迭代中积累了大量技术债，可靠性难以保障。
    

## 三、什么是“OpenClaw养虾”？

“养虾”是中文互联网对**部署、配置并持续使用OpenClaw这一过程**的形象比喻。就像饲养一只宠物虾需要准备环境、投喂食物、训练技能一样，“养虾”也包含了类似步骤：

- **搭建“虾缸”**：准备硬件和软件环境，安装OpenClaw。
    
- **投喂“虾粮”**：为其配置和接入大模型API，赋予其“大脑”。
    
- **安装“技能”**：从社区获取或自己开发技能，扩展其能力。
    
- **日常“驯化”**：通过不断下达指令，让它学习你的工作习惯和偏好，越用越聪明。
    

上海财经大学教授胡延平指出，OpenClaw基本上实现了对设备和人的“接管”，已接近下一代操作系统的雏形。“养虾”的本质，就是让个人电脑化身为一个不知疲倦、高度个性化的“数字员工”。

## 四、如何“养虾”？——部署与使用指南

“养虾”主要有**本地部署**和**云端部署**两种方式。

**对于技术新手，推荐更易上手的简化流程：**

1. **环境准备**：一台Windows 10/11或macOS电脑，能正常上网，并准备好微信、飞书或QQ等常用聊天软件。
    
2. **下载安装**：从GitHub下载OpenClaw的最新版安装包，双击运行，按照提示完成安装。
    
3. **绑定通讯工具**：安装完成后，OpenClaw界面会生成一个二维码。用你的微信/飞书/QQ扫码登录，完成绑定。
    
4. **测试与使用**：绑定成功后，即可在聊天软件中向它发送指令。例如，发送“帮我新建一个txt文件，并写入‘养虾成功’”，如果电脑自动执行了该操作，则说明部署成功。
    

**对于追求深度定制的用户，则需遵循更专业的步骤：**

- 安装Node.js等运行环境，通过命令行进行部署。
    
- 自行申请并配置OpenAI、Claude等大模型的API密钥。
    
- 通过社区技能市场或自行开发来扩展功能。
    

无论哪种方式，都需要注意控制API调用成本，并从可信来源获取技能，以规避安全风险。

## 五、OpenClaw vs. WorkBuddy：毛坯房与精装房之争

随着OpenClaw爆火，腾讯于2026年3月推出了对标产品**WorkBuddy**。两者使命相似，但产品哲学和实现路径截然不同。

|对比维度|OpenClaw|WorkBuddy (腾讯)|
|---|---|---|
|**产品定位**​|开源框架，高度自由，社区驱动|自研产品，安全可控，生态整合|
|**部署门槛**​|极客级，需命令行、环境配置|普通人级，下载安装即用，1分钟完成配置|
|**自动化程度**​|追求高度自主化，可持续执行任务|设置人工确认环节，控制在“半自动”区间，不替用户做决定|
|**平台接入**​|需自行对接各种通讯工具|原生支持企业微信、QQ、飞书、钉钉等|
|**安全与合规**​|用户自行负责，风险高|强调企业级安全、审计与风险拦截|
|**技能生态**​|依赖社区，自由但需自行筛选|内置20+办公场景技能，并兼容OpenClaw社区技能导入|
|**核心用户**​|开发者、技术极客、喜欢折腾的用户|普通办公人员、企业用户、追求开箱即用的群体|

用形象的比喻来说：**OpenClaw是“毛坯房”**，它提供了强大的地基和框架，但装修、通水电都得自己来，适合喜欢DIY和拥有完全控制权的用户。**WorkBuddy则是“精装房”**，家具家电一应俱全，拎包入住，主打安全、易用和与腾讯生态的深度整合，目标是让AI助手真正成为普通人的办公伙伴。

WorkBuddy负责人汪晟杰强调，该产品是百分之百自研，并非OpenClaw的套壳。其设计理念是“帮用户做繁琐的事，但不替用户做决定”，在自动化与可控性之间找到了平衡点。

## 六、总结

OpenClaw的兴起，标志着AI从“对话与建议”走向“感知与执行”的关键一步。它点燃了“个人AI代理”的赛道，让普通人看到了拥有一个专属数字助手的可能性。然而，其伴随的高风险和高门槛，也决定了它目前仍是技术爱好者的前沿玩具。

腾讯WorkBuddy等产品的出现，则代表了另一种思路：通过降低使用门槛、加强安全管控、聚焦办公场景，将AI智能体技术推向更广阔的大众市场。这场“毛坯房”与“精装房”的竞争，本质上是对未来人机协作形态的不同探索。

无论选择哪条路径，“养虾”热潮背后，是人们对解放生产力、让机器承担重复性工作的共同渴望。随着技术不断成熟和安全方案的完善，真正智能、可靠且易用的“数字员工”走进千家万户的办公桌，或许已不再遥远。

什么是文件路径？

路径即文件或文件夹在电脑中的位置，示例：
- **Windows**：`C:\Users\你的用户名\Desktop\工作资料`
- **Mac**：`/Users/你的用户名/Desktop/工作资料`

# 结果查看

## 一、右侧结果区

任务执行过程中，右侧会显示结果区，用来查看 WorkBuddy 产出的文件、代码变更和预览内容。您不需要离开当前任务，就可以一边看对话，一边查看结果。

当前结果区主要提供四类内容：

- **产物**：查看任务生成的文件和交付物
- **全部文件**：查看当前工作空间中的文件内容
- **变更**：查看本次任务带来的文件修改
- **预览**：查看网页或可直接打开的结果页面

这些内容都集中在同一块区域中，方便您来回切换和对照。

## 二、产物

当任务生成文件、表格、文档或其他可交付内容时，您可以在 **产物** 中统一查看。

产物区通常包含两部分：

- 左侧是当前任务生成的产物列表
- 右侧是选中产物的预览区域

# 连接器
一句话
连接器用于将 WorkBuddy 与外部服务进行对接，实现数据互通与能力扩展。通过配置连接器，您可以让 AI 助手直接访问和操作第三方服务。
## 一、什么是连接器
连接器是 WorkBuddy 与外部系统之间的桥梁。它基于标准化协议（如 MCP）将外部服务的能力引入到 AI 工作流中，让您无需手动切换工具即可完成跨平台操作。

| 场景       | 说明             | 示例          |
| -------- | -------------- | ----------- |
| **数据查询** | 从外部数据源获取信息     | 查询数据库、检索文档  |
| **服务调用** | 调用第三方 API 完成操作 | 发送邮件、创建日程   |
| **文件管理** | 访问云端存储与文件系统    | 读取网盘文件、上传附件 |
| **消息通知** | 与即时通讯工具集成      | 发送企业微信/飞书消息 |
## 二、进入连接器管理
在左侧导航栏中点击 **连接器**，即可进入连接器管理页面。页面展示所有可用的连接器，当前支持 **QQ邮箱** 和 **腾讯乐享**，并支持自定义连接器。
![](assets/WorkBuddy入门到精通/file-20260505192752953.png)
## 三、连接 QQ 邮箱
QQ 邮箱连接器支持收发、搜索和整理 QQ 邮件，用自然语言读取邮件内容、汇总邮件线程、管理文件夹。
### 1）点击添加
在连接器管理页面找到 **QQ邮箱** 卡片，点击右侧的 **+** 按钮发起连接。
![702](assets/WorkBuddy入门到精通/file-20260505192806154.png)
### 2）扫码授权
系统会跳转到 QQ 邮箱授权页面，显示二维码。使用 **QQ 邮箱 App**（7.1.5 / 鸿蒙 0.2.9 及更新版本）扫码进行授权。
![829](assets/WorkBuddy入门到精通/file-20260505192815637.png)

### 3）确认权限
在手机端确认 WorkBuddy 申请的访问权限，包括：
- **账号信息**：获取邮箱账号基础信息
- **读取邮件**：读取收件箱内近一个月的邮件
- **发送邮件**：代为发送邮件
- **删除邮件**（可选）：已删除邮件会移入「已删除」文件夹
确认无误后点击 **同意授权**。
![403](assets/WorkBuddy入门到精通/file-20260505192822304.png)

### 4）授权成功
手机端显示 **授权成功**，可在「设置 - 账号 - 安全管理 - 登录设备和授权码 - 应用授权」中管理授权应用。
![389](assets/WorkBuddy入门到精通/file-20260505192835471.png)

### 5）连接完成
回到 WorkBuddy，QQ 邮箱卡片名称旁会出现绿色圆点，并提示 **连接器 QQ邮箱 已连接**，右侧显示启用/禁用开关。
![758](assets/WorkBuddy入门到精通/file-20260505192843037.png)
![749](assets/WorkBuddy入门到精通/file-20260505192902340.png)

### 1）点击添加
在连接器管理页面找到 **腾讯乐享** 卡片，点击右侧的 **+** 按钮发起连接。
![728](assets/WorkBuddy入门到精通/file-20260505192910221.png)
### 2）登录乐享知识库
系统会跳转到腾讯乐享登录页面，支持 **微信登录**、**手机号登录** 和 **企业微信登录** 三种方式。
![751](assets/WorkBuddy入门到精通/file-20260505192915689.png)

### 3）确认授权
登录后进入授权确认页面，显示当前账号信息。通过乐享知识库 MCP，WorkBuddy 将能够：

- 按你拥有的权限访问知识库
- 搜索、查询知识库中的内容
- 协助创建与管理知识库中的内容
确认无误后点击 **继续**。
![792](assets/WorkBuddy入门到精通/file-20260505192922471.png)

### 4）授权成功
页面显示 **Authentication Successful**，表示已成功完成认证，可关闭当前窗口返回 WorkBuddy。

### 5）连接完成

自定义连接器的配置方式与 MCP 配置类似，详细说明可参考 [MCP 配置指南](https://www.codebuddy.cn/docs/workbuddy/From-Beginner-to-Expert-Guide/Function-Description/MCP-Guide)。

### 2）确认断开

### 3）断开完成

![](assets/WorkBuddy入门到精通/file-20260505192958188.png)

## 七、注意事项
安全提示
- 连接器需要相应服务的访问权限，请确保授权信息正确
- 授权后可随时通过开关禁用或断开连接器
- 如需撤销 QQ 邮箱授权，可前往 QQ 邮箱 App「设置 - 账号 - 安全管理 - 应用授权」中操作
- 建议定期检查连接器状态，确保服务正常运行

- **对话入口联动**：对话界面的模型选择器展示自定义模型分组，并支持快捷跳转到配置界面进行编辑。

配置在保存后自动持久化，选择标准供应商时，工具调用、图片输入等能力标记会自动写入，无需手动配置。
已通过 `~/.codebuddy/models.json` 配置的自定义模型在界面升级后仍可正常使用，并可通过 UI 界面查看、编辑或删除，无需再直接操作配置文件。

### 接入方式
#### 提供商接入
从提供商列表中选择或选择自定义API后，URL、模型列表、能力标记（工具调用、图片输入、推理模式等）会自动填充，仅需补全 **API Key** 即可保存使用，全程无需手动修改任何配置文件。
- **Token Plan**
需要按 Token 用量计费的团队，可在自定义模型配置中选择腾讯云 Token Plan 作为鉴权方式，支持混元、MiniMax、Kimi 等多款主流模型。

#### 本地部署
**Ollama** 是一个开源的本地大模型运行工具，安装后通过一行命令即可拉取并运行开源模型。Ollama 启动后会在本地监听 HTTP 端口（默认 11434），并自动提供兼容 OpenAI 协议的接口供WorkBuddy对接。

**适用场景：**
1. 数据隐私 / 内网合规：代码与对话内容不出本机，适合金融、政企、敏感项目场景。
2. 零成本试用：不用消耗 Token、不用付费 API Key，把硬件当算力。
3. 离线可用：飞机上、内网开发机、没网的环境也能用 WorkBuddy。
![524](assets/WorkBuddy入门到精通/file-20260505193127339.png)

如果你的模型服务不在上方列表中，可选择**自定义/Custom**，手动填写 URL、API Key 与模型名接入。

### 自定义协议
当模型服务使用非标准 URL 路径（如经过网关或代理层封装）时，可在高级配置中开启**自定义协议**开关。开启后 WorkBuddy将直接按填写的 URL 发起请求，跳过路径校验与自动补全。

|状态|行为|
|---|---|
|关闭（默认）|使用标准 `/chat/completions` 路径，自动校验并补全接口地址|
|开启|直接使用用户填写的接口地址发起请求，跳过路径校验与自动补全逻辑|
